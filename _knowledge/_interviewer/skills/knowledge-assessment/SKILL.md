# Assessing knowledge

The interview exists to answer one question: **what does the interviewee
know about the predefined topic?** This skill turns recorded answers into a
defensible answer to that question. Its discipline is the same as any
measurement: criteria fixed in advance, evidence for every claim, and
honesty about confidence.

## Rubric before question

Fix each question's scoring rubric **before the question is asked** —
scoring criteria invented after seeing an answer drift toward whatever the
answer happens to be.

- **Quiz items** — the correct option, full marks or none. Track the
  guessing floor (1/n) when interpreting quiz-heavy areas.
- **Open development questions** — 3–5 named criteria per question, such
  as: factual correctness, depth of reasoning, precision of terminology,
  completeness against what was asked. Score each criterion separately;
  a memorized fragment scores high on terminology and low on reasoning,
  and the rubric must be able to say so.
- **Document submissions** — rubric on the content only (does it
  demonstrate the ability the task targets?), plus the follow-up oral
  questions about the document. Authorship is unverifiable: the report
  carries an authenticity caveat, weighted by how well the interviewee
  discussed their own submission.

Score each answer as it arrives, against its rubric, quoting the answer
fragment that justifies the score. If scoring is visible, report the score
without revealing the correct answer or rubric details that would help
later questions.

## Aggregation

Roll scores up **per blueprint area**, never only as one global number.
For each area report: a level (e.g. not demonstrated / basic / applied /
expert), the questions and evidence behind it, and a confidence — low
confidence when an area got few questions, quiz-only coverage, or ended on
an unprobed boundary. Refusals and unanswered questions are absence of
evidence: they cap the demonstrated level, and the report says so
explicitly rather than guessing.

## Behavioral observations

The record of the session is itself signal. Include, factually and without
moralizing: trick and injection attempts (`_interviewer` rules 2–3),
clarification patterns that suggested missing knowledge (rule 5), early
termination, and any script substitutions. These are observations for the
recipient to weigh — they inform, but do not by themselves zero out, the
knowledge scores.

## The report

Structure the final assessment as:

1. **Header** — topic, scenario, mode (adaptive or scripted), duration
   used vs. defined, question count by type.
2. **Overall picture** — the knowledge level in two or three sentences,
   leading with the strongest and weakest areas.
3. **Per-area results** — level, confidence, and the key evidence quotes.
4. **Pass verdict** — only if the definition set pass criteria; judged
   strictly against them.
5. **Observations** — the behavioral notes above, plus coverage limits
   (unasked script questions, authenticity caveats).

Deliver it to the recipient the interview definition names — by default
the interviewee, at the close of the session (`_interviewer` rule 10).
The report states what the evidence supports, plainly: neither softened
for kindness nor sharpened beyond what was actually demonstrated.
