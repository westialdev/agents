Main project rules
==================

1. Strictly follow the Development Guidelines and Done Skills for every task involving changing the code.
2. Look at the doc/UBIQUITOUS.md for the Ubiquitous Language, and so, we can all agree on the same terms.
3. When a new concept, crucial for the domain, is introduced, update the doc/UBIQUITOUS.md with the new term.
4. When starting a new task, first check the README.md. Do not check it if you already did it.
5. After delivering a task and getting the developer approval, update the README.md with the new task you are working on.
